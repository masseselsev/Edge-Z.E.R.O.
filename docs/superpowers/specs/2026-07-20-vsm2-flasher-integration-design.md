# Specification: VSM2 Flasher Integration

This specification details the design for porting the `mass_flasher` utility into the Edge-Z.E.R.O. Overwatch hub (`overwatch`). The integration will be implemented using FastAPI for the backend and React + Tailwind CSS for the frontend, introducing a new "VSM2 Flasher" tab.

---

## 1. Settings Consolidation & Alignment

To avoid duplication, VSM2 Flasher will align with and reuse existing `overwatch` system configuration keys from the `system_settings` database table:
* **Telegram Notifications:** Reuses `TELEGRAM_BOT_TOKEN` and `TELEGRAM_CHAT_ID`.
* **Timezone:** Reuses `DEFAULT_TIMEZONE` (defaults to `"UTC"`).
* **Locale:** Automatically synced from the frontend active language state (`en`, `uk`, `ru`).
* **SSH Defaults:** VSM2 Flasher defaults (SSH Username, Password, Port) will be managed via the flasher inputs but can fallback to default box credentials if available.

---

## 2. Backend FastAPI Architecture

We will implement a new router in `backend/app/api/endpoints/vsm2_flasher.py` mounted at `/api/vsm2-flasher`:

### Git Repository Cache
* **Repository:** `https://github.com/masseselsev/controlboard.git`
* **Local Path:** Cloned into `/app/vsm2_repo_cache` (mapped to `./backend/vsm2_repo_cache` on host, ignored in `.gitignore`).
* **Startup Sync:** On API startup, a background task will run git clone or git pull (fetch & hard reset to `origin/main`) to ensure we have the latest setup script and `.hex` firmware files.
* **Static File Server:** An endpoint `GET /api/vsm2-flasher/files/{filename:path}` will serve files directly from the repository cache, allowing target boxes to download `setup.sh` and firmware binaries.

### Flashing Engine (Async Thread Workers)
* **Start Flash Endpoint:** `POST /api/vsm2-flasher/flash`
  * **Payload:** Target IP string (e.g. `192.168.1.10-20`), SSH username, password, port, and advertised flasher IP.
  * **Flow:**
    1. Parses IP ranges and checks if any target is already in `ACTIVE_TASKS` to prevent duplicate concurrent flashes.
    2. Triggers an online check/repo sync.
    3. Spawns a Python thread running `FlashWorker` (re-implemented from `ssh_utils.py` using paramiko or asyncssh/asyncio equivalent) for each target.
    4. The worker connects via SSH, sets environment variables (including local flasher `BASE_URL`), downloads `setup.sh`, and runs it with `--flash-cleanup`.
    5. Returns immediately with starting status.
* **Server Logs Stream (SSE):** `GET /api/vsm2-flasher/stream`
  * Streaming endpoint returning `text/event-stream` logs generated by active workers, allowing the UI to show live console logs for each device.
* **Clear Logs:** `POST /api/vsm2-flasher/logs/clear` clears server-side log history.

### Console REPL & Interactive Shell
* **Check/Connect Console:** `POST /api/vsm2-flasher/console/connect`
  * Establishes SSH connection to the target device.
  * Checks if controlboard scripts are present on target; if missing, uploads `app.py`, `dist/commands.py`, `dist/controlboard.py`, sets up a virtual environment, installs dependencies, and adds the user to the `dialout` group.
  * Spawns an interactive REPL shell (`app.py` on the target device) on a background thread/task.
* **Send Console Command:** `POST /api/vsm2-flasher/console/send`
  * Sends a command to the target device's active REPL session.
* **Disconnect Console:** `POST /api/vsm2-flasher/console/disconnect`
  * Closes the target device REPL session.
* **Available Commands list:** `GET /api/vsm2-flasher/console/commands`
  * Parses commands from the cached `dist/commands.py` file to provide auto-complete options in the frontend.
* **Batch Read Parameters Dump:** `POST /api/vsm2-flasher/console/batch_read`
  * Performs sequential reads of multiple Modbus registers on the target controller and returns a JSON report of all parameter values.

## 3. Frontend React Component

A new tab component `frontend/src/components/Vsm2FlasherTab.tsx` will be created:

### Tab Layout (Grid System)
* **Left Panel - Targets & SSH Credentials:**
  * Input for target IPs/ranges (with tooltip showing supported formats: comma, spaces, ranges).
  * SSH Credentials (User, Password, Port).
  * Advertised Flasher IP (with auto-detect option showing local interface IPs).
  * "Start Mass Flash & Reboot" primary action button.
  * Repository Status card showing current cached commit hash, author, branch, and last sync timestamp.
* **Right Panel - Controller Console & Logs (Sub-tabs):**
  * **Console Sub-tab:**
    * Target IP selection, Connect/Disconnect button.
    * Command input field supporting autocompletion (populated by `/api/vsm2-flasher/console/commands`).
    * Interactive CLI terminal window showing raw output.
    * Quick Actions config: grid of customizable buttons.
    * Batch Parameters Dump: Modal dialogue allowing users to select registers (e.g. Temperature, Version, Humidity) to read in a batch, displaying a table of results with "Copy to Clipboard" and "Download .txt" actions.
  * **Live Logs Sub-tab:**
    * Scrolling output display showing flashing progress of all devices (fed by SSE `/stream`).
    * "Clear Logs" action button.

## 4. Navigation & Translation Changes

* **Tab Insertion:** The "VSM2 Flasher" tab will be added between "Settings" and "Edge-B.R.O." in `App.tsx` and `Header.tsx`.
* **Tab Renaming:** The "Edge B.R.O." tab is renamed to "Edge-B.R.O." in:
  * `frontend/src/components/Header.tsx`
  * `frontend/src/i18n/translations.ts`
* **Translation Additions:** The new localization keys (`tabVsm2Flasher`, `vsm2FlasherTitle`, etc.) will be added in `translations.ts` for English, Ukrainian, and Russian.

## 5. Verification Plan

* **Automated Tests:**
  * Validate parsing of IP ranges (unit test for parser logic).
  * Mock SSH connections to ensure the bootstrap script uploads and virtual environment creation flows function correctly.
* **Manual Verification:**
  * Verify UI alignment and tab switching.
  * Confirm that the Telegram alert is dispatched correctly when a flashing thread finishes.
  * Ensure that logs stream smoothly without freezing the browser tab.
